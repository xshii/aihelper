## 2. 10 分钟 Hello World：跑一个最简用例

### 2.1 前提

- 已加入项目 git 仓、有 CMC 制品仓访问权限（见 00 · 3.3）
- 本地环境通过 `env_check.sh` 自检（见 00 · 3.4）
- 项目默认平台是 FPGA，本 Hello World 也跑在 FPGA 上

### 2.2 用例代码（`tests/hello/test_hello_world.py`）

```python
import pytest

@pytest.mark.accuracy
@pytest.mark.l0
def test_hello_resnet50_cpu_baseline(fpga, case_loader):
    """最简精度用例：跑 resnet50 的 CPU 简化实现（首用例规格），bit 级比对。"""
    case = case_loader.load("cv_resnet50_cpu_baseline")

    # 1. 加载 DO（业务软件 + 模型）
    fpga.load_do(case.do)

    # 2. 选用例 + 注入激励
    fpga.select_case(case)
    fpga.inject_stimulus(case.stimulus)

    # 3. 触发业务激励 + 等待完成
    fpga.trigger()
    fpga.wait_until_done(timeout_s=120)

    # 4. 精准比数（框架自动走"标准路径"或"备选路径"）
    verdict = fpga.compare(case.golden)

    # 5. 裁决（bit 级一致，见 02 · 7.1）
    assert verdict.is_pass(), verdict.explain()
```

### 2.3 运行

```bash
# 单用例
pytest tests/hello/test_hello_world.py -v

# 带 HTML 报告
pytest tests/hello/test_hello_world.py --html=report.html --self-contained-html

# 只跑 L0 冒烟
pytest -m "l0 and accuracy" -v
```

### 2.4 输出（期望）

- 终端：`test_hello_resnet50_cpu_baseline PASSED`
- `report.html`：用例详情 + bit 比对摘要
- 归档目录 `reports/{date}/{build}/cv_resnet50_cpu_baseline/`：Autotest 日志 + compare 报告 + DFX 快照

### 2.5 失败了怎么办

| 错误 | 最可能原因 | 看哪里 |
|---|---|---|
| `fpga` fixture 起不来 | 板级资源忙 / 加载失败 | 00 · 4.4.5 + 01 · 2.11 |
| VerFingerPrint 不匹配 | 组件版本错配 | 00 · 4.3 + 01 · 2.10 |
| compare FAIL | 真有 bit 不一致 | 02 · 7.10 综合定位建议 |
| 拉不到 GOLDEN | dbox / onebox 通道问题 | 03 · 2.2.4 |
| DFX 告警 | 硬件侧异常 | 04 |

## 5. 第一周常见问题

| 问题 | 答 |
|---|---|
| 文档这么多，必须全部读完才能开始吗？ | 不必。按第 4.1 节 Day 1~2 读完就能跑用例；其他按需查 |
| 为什么精度是 bit 级一致，不是容忍度？ | 硬件验证本质约束；详见 [ADR-02](./10_子wiki_ADR决策记录.md)（和 02 · 7.1）|
| GOLDEN 为什么不能用 PyTorch 生成？ | 跨平台 bit 级一致要求；详见 [ADR-03](./10_子wiki_ADR决策记录.md)（和 03 · 2.2）|
| 性能用例为什么跑 FPGA 是"参考"而 EMU 才是"权威"？ | FPGA 频率受综合约束不准；详见 05 · 1.2 和 03 · 3.6 |
| 我写的用例要跑几遍？ | 默认 N=5 取中位数，首次 warmup 不计入（03 · 3.5）|
| 一个用例要多长？ | 用例级超时默认按套件配置；阶段级用 `@stage_timeout` 自定义；详见 Autotest 项目文档 |
| markers 要怎么加？ | 每个用例至少 1 个用例类型（`accuracy/perf/functional/stability`）+ 1 个级别（`l0/l1/l2/l3`）；见 06 · 8.8 |
| 首用例是干嘛的？必须有吗？ | 每个精度用例集**至少 1 个**首用例；作为硬件优化开关的"零点"，见 02 · 5.1 |
| 碰到没写在文档里的问题去哪问？ | 先查各 wiki 的"常见问题"小节；再在项目 chat 频道问；最后交由 triage（见 10 ADR-05 + 失败路由规则）|
| 自己跑的回归和 CI 跑的结果不一样 | 优先检查 baseline / GC 版本是否对齐（00 · 4.3 VerFingerPrint）；其次检查资源调度（LAVA 分配的板）|
